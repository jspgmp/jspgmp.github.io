20211111-KDT-Worksheet

Posted on 2021-11-11 In SPRING Disqus:

  1. Inversion of Control

  • 객체를 스프링 컨테이너가 만들게끔 하는 방식
  • 별도 xml 설정 필요

  1. Aspect Oriented Programming

  • 비즈니스 로직인 핵심 관심과, 그 전후 Advice인 횡단 관심을 결합하는 것 = Weaving
  • 핵심 관심에 Pointcut을 지정하고 이것을 Advice와 연결하는 형태 = Aspect
  • Advice 의 동작 시점은 크게 3가지로 구분된다.
    • Before : 실행 전
    • After
      • After Returning : 성공적 실행 후
      • After Throwing : 예외 발생 시
      • After : 무조건 실행
    • Around : 실행 전/후

  1. 설정

유지보수 과정에서 객체가 자주 변경된다면 등록을,

자주 변경되지 않는다면 Annotation으로 처리한다.

  1. 서비스 구조(게시판)

Annotation 을 사용할 경우, Component-scan 을 xml 에 등록하여 각 java파일에서 @가 스캔될 수 있도록 한다.

  • Board Value Object : 하나의 게시글에 대한 틀과 기본 요소

  • Board Data Acess Object Interface : 구현방식 변경을 위한 인터페이스

    ​ @Repository 등록을 하거나 등록

    • Board Data Acess Object (JDBC) : JDBC 방식으로 구현한 클래스(세부내용)
    • Board Data Acess Object (SPRING) : SPRING 방식으로 구현한 클래스(세부내용)
      • Board Row Mapper : SPRING 방식이 사용하는 방식에서 필요한 클래스
      • SPRING 방식을 이용할 경우 dataSource 및 Transation 설정을 xml 에 등록해야한다.

  • Board Service Interface : 비즈니스 로직 변경을 위한 인터페이스

    ​ @Service(“boardService”) //다른 로직으로 변경할경우 이 서비스 부분을 옮긴다.

    • Board Service Impl : 컨테이너와 닿는부분이며, 실제 비즈니스로직의 실행을 안내한다.

  1. Advice 구조

Advice 들은 파일 위치보다는 Annotation을 따라가므로 알아서 분류를 한다.

Advices 라는 파일에 전부 모여있다고 가정을 하면

​ @Aspect

  • BoardPointcut : 핵심관심에 Point를 달아줘야 Advice가 붙을 수 있다.

    ​ @Pointcut(주소)

    • Public void allPoint 또는 getPoint

  • Advices : Advice들의 모음

    Pointcut 등록은

    ​ @Before(“BoardPointcut.allPointcut()”)

    • BeforeAdvice method

      @AfterReturning(상동.getPointcut())

    • AfterReturningAdivice method

      @AfterThrowing(상동)

    • AfterThrowingAdvice method

      @After(상동)

    • After method

      @Around(상동)

    • aroundLog method

실제 비즈니스 로직의 구동방식 변경, 로직 모델 변경을 위해 2개의 인터페이스를 쓰고있다는 점과

Advice들을 실제 비즈니스 로직의 사이에 Aspect 를 붙여 연결하여

중요한 메서드 실행 전/후 처리를 이해한다.
