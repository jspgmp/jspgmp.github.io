스프링부트로 JPA 이용하기 적용하기

설정

  1. VO에 @Entitiy설정

    식별자 primary key 에 @Id, @GeneratedValue 설정

    업데이트로 값이 변경되는 안되는 값에 @Column(updatable = false) 설정

    작성날짜의 경우 @Column(insertable=false, updatable=false, columnDefinition = “date default sysdate”)

    조회수의 경우 @Column(insertable = false, updatable=false, columnDefinition=”number default 0”)

    searchCondition, keyword 와 같은 경우 컬럼과 연결하지 않으므로

    @Transient

    private String searchCondition ;

    과같이 설정한다.

  2. persistance 패키지에 Repository 인터페이스 생성 (DAO 클래스와 동일한 개념의 인터페이스)

    인터페이스만 정의하면 구현 클래스는 스프링부트가 자동으로 제공한다.

    extends CrudRepositry<VO, ID로 지정한 변수의 데이터 타입>

    @Repository 설정,

    MyBatis와 차이 : CRUD 메서드를 일일히 구현하지 않고 2번 설정만 하면 자동으로 제공한다.

    MyBatis와 차이 : SQL을 제공해준다.

    • 주요 메서드 : findAll(), save(), delete(), findById() 등

    • update는 없으므로 조회후 수정정보로 다시 저장 하는 과정 필요

    • 조회 결과가 있을수도있고, 없을수도없다면

  3. ServiceImpl 에 Repository를 @Autowired하고 CRUD설정한다.

  1. 패키지 설정
  • domain에 vo를, persistance에 Repository를, service에 service 와 serviceImpl을 넣는다.

JPQL

  1. JPA Query Language

Repository 인터페이스에 설정하며,

@Query(“JPQL”) + 메서드 형식이다.

SQL과 형식이 유사하나, FROM 절 뒤는 테이블이 아니라 Entity 클래스이다.

JPQL을 실행하면 HIbernate에 의해서 H2 DB에 맞는 SQL로 변환되어 실행된다.

JPQL을 자동으로 만들어주는 Query메서드가 제공된다.(findBy~)

메소드 유형

  1. Native SQL : 원래 SQL을 사용할 때 nativeQuery 사용

일부 컬럼만 조회하는 경우 Object[] 자료형으로 반환하도록 한다.

Page

  1. pageable 인터페이스

    검색된 데이터들을 지정한 단위 페이지로 쪼개어 나타내준다.

    조회 대상의 자료 형식을 Page로 설정하고 JPQL메서드 두번째 인자로 pageable을 넣으면 된다.

  2. 실질적인 글 목록 정보는 getContent()로 얻어낸다.

Security

설정

  1. pom.xml에 addtarter 하여 spring security를 추가한다.

    초기 Username 은 user이다.

    config 패키지에 SecurityConfiguration extends WebSecurityConfigurerAdapter 클래스 생성

  2. ```java
    @Autowired // 3번 클래스 의존성주입
    private UserDetailsServiceImpl userDetailsService;

    //CSRF 토큰을 사용하지 않도록 설정
    http.csrf().disable();
    // 특정 경로에 대한 접근 제어 설정
    http.authorizeRequests()
    // 특정 경로에 대한 접근 허용
    .antMatchers(“경로”).permitAll()
    // 특정 경로에 대한 접근 제한(로그인)
    .antMatchers(“경로”).authentificated()
    // 특정 옵션을 가진 자에게만 오픈
    .antMatchers(“경로”).hasOption(“옵션”);
    // 시큐리티 기본 로그인 화면을 로그인페이지로 설정
    http.formLogin().loginPage(“로그인페이지”).defaultSuccessUrl(“로그인성공시이동페이지”);
    // 로그아웃 요청시 세션 종료 및 인덱스페이지로 이동
    http.logout().logoutUrl(“로그아웃”).invalidateHttpSession(true).logoutSuccessUrl(“/“);

    // 기본설정이 아닌 내가만든 클래스 멤버 사용 설정
    http.userDetailsService(userDetailsService);
