Posted on 2021-12-15 In JAVA SPRING Disqus:

스프링 부트로 MyBatis 이용하기

설정

JDBC 연결로 설정된 DAO를 MyBatis로 전환하기 위한 방법이다.

  1. DAOMyBatis를 설정하기
    • 클래스에 @Repository를 붙여 Bean 등록하여 검색되도록 한다.
    • 변수에 @Autowired를 달고 private SqlSessionTemplate mybatis 추가
    • CRUD 메서드 설정
    • 연결된 Service 및 ServiceImpl 에 참조된 변수를 DAOMyBatis로 변경한다.
  2. mapping.xml 설정하기
    • mapper 설정
    • 결과를 담을 resultMap 설정
    • SQL을 담는 select 설정 (id, parameterType, resultMap을 포함한다. )
    • select id 는 각각의 CRUD 메서드 에서 가르키게 된다.
  3. application.properties 설정
    • Alias에 해당하는(VO) 클래스가 들어있는 패키지 지정(VO를 한곳에 모아야 한다.)
      • mybatis.type-aliases-package=
    • MyBatis가 사용할 mapper XML 파일들의 위치 지정(resource가 기본 경로인듯하다. )
      • mybatis.mapper-locations=

후기

JDBC로 진행하며 초기 DB 테이블에 모자란 컬럼을 억지로 조인트하여 결국

JDBC DAO 설정 시 매개변수를 억지로 지정하는 바람에 MyBatis 사용 시 어려움이 있었다.

MyBatis에서는 하나의 매개변수만 받는 듯 하다.

예: mybatis.insert(“insert”, AVO, BVO) 이런식은 안됌.

그래서 테이블 자체를 수정하여 테이블간 조인트를 줄이는 방법으로 수정했다.

  • Mapper Interface

    @Mapper를 달고 @Select (SQL문)을 달고

    어떤 매개변수를 담을지 어떤 자료형을 반환할지 까지 지정한다.

  1. 변형

    • Mapper 자바 파일에서 SQL을 전부 뺴기위해서 application.properties에 Alias와 MyBatis 위치를 지정하고 Mapper interface를 지운다.

    • 매핑 xml 작성시

      mapper namespace=Mapper 주소

      resultMap type = VO클래스명

      id = 메서드명

      parameterType = VO클래스명을 잘 지켜야 오류가 안난다.
