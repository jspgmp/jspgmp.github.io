시스템 프로그래밍

리눅스

서버에 많이 사용

  • 최근 서버 환경 대다수
  • 클라우드 컴퓨팅에 사용
    • 예 : 아마존 aws
  • 프로그래밍에 유용
  • 유닉스 계열 운영체제

권한

  • 모든 리소스를 파일로 관리(inode)
  • 사용자와 그룹으로 권한 관리

ANSI-C

  • 가장 많이 사용하는 C언어 표준
  • 영문 알파벳만 사용할 수 있음

클라우드 컴퓨팅 설정

  1. EC2 서버 설정
  2. Elastic IP 생성
  3. 자기 PC에서 EC2 접속

쉘 종류

  • Bourne -Again Shell(bash) : GNU 프로젝트 일한으로 개발된 리눅스 디폴트
  • Bourne Shell(sh)
  • C shell(csh)
  • Korn Shell (ksh) : 유닉스에서 가장 많이 사용

기본 명령어

  • whoami: 로그인한 사용자 ID를 알려준다.
  • passwd: 로그인한 사용자 ID의 암호 변경
  • useradd: 사용자 기본 설정 자동으로 안함
  • adduser: 사용자 기본 설정 자동수행
  • su: 사용자 변경
    • su root: 현재 사용자의 환경설정 기반을 root으로 변경
    • su - root: 현병될 사용자의 환경설정 기반으로 root전환
  • pwd: 현재 디렉토리 위치
  • cd: 디렉토리 이동
  • ls: 파일 목록 출력
  • man: 사용법
    • 예: man rm
  • chmod: 파일 권한 변경
  • chown: 소유자 변경
    • chown 옵션 소유자:소유그룹 파일
  • cat: 파일 보기
  • head/tail: 시작이나 끝 부분
  • more: 파일보기
  • rm: 파일 및 폴더 삭제

sudo 명령어

root 계정에 로그인하지 않은 상태에서, root 권한이 필요한 명령을 실행할 수 있도록 함

  • 예: sudo apt-get update
