MVC

Model + Controller + View

Model : 비즈니스 모델 = > (DAO, VO)

Controller : 비즈니스 모델을 컨트롤하는 역할

  • 컨트롤러는 뷰에서 자바 코드만 빼서 어떤 모델을 어떤 컨트롤러가 처리할지 매핑
  • 컨트롤러는(자바개발자 또는 MVC 프레임워크 사용)
  • 처리하고나면 뷰로 String 매핑과정에서 서블릿은 DispatcherServlet 하나

View: 실제 보여지는 화면에 분배

자바 소스와 디자인을 수정하기 어려워서(화면 HTML과 자바 소스가 뒤섞임) 나오게 된 것으로 dispatcherservlet 을 이용해서 서블렛과 디자인을 분리시킴

EL / JSPL 소개와 적용

EL : ${} 로 구성되어 HTML사이에 자바 요소를 인식할 수 있게 해준다.

JSPL : <%= %>로 구성되어 HTML 사이에 자바 수식을 인식할 수 있게 해준다(이것은 컨트롤러로 넘어가게 됌)

XML

  • 등록가능: 서블릿, 필터, 리스너

  • 레이어를 두개로 분리 (presentation, business)

작동 순서

톰캣이 서블릿 컨테이너 생성

디스패처 서블릿이 스프링 컨테이너 생성

구성: 서블릿 컨테이너 하나+스프링 컨테이너 2개

  1. ContextLoader 프리로딩

  2. 첫번째 컨테이너 생성

  3. business layer 로드

  4. 서비스 임플, dao 로딩

  5. 클라이언트가 .do 요청 후

  6. dispatcher가 레이지 로딩

  7. xmlwebapp 두번째 객체를 presentation-layer를 로딩해서 business Layer 를 참조함

예전에는 컨트롤러까지 전부 개발해서 썼지만 요즘에는 외부 컨트롤러를 사용하는 경우가 많다.

Spring Boot: Starter

메이븐: 프로젝트 라이브러리 관리 툴

  1. @Bean 사용법

    • @Bean 이 설정된 메서드는 자동으로 호출된다.
    • @Bean에 의해 생성된 객체의 아이디는 메서드 이름이 자동으로 설정
    • 예: 컨테이너 실행시 Bean 이 붙은 메서드 호출
    • 스프링부트는 스프링에서 제한한 자바코드 수정 규칙을 깸으로 얻는 장점이 있다.

  2. @Configuration 사용법

    • @Bean 이 붙은 Class에 붙여서 사용

  3. AnnotationConfigApplicationContext 사용법

    • XML을 사용하지 않고 설정 클래스를 사용한다.

    • 매개변수로 지정한 패키지를 기준으로 @Configuration이 붙은 클래스를 모두 로딩한다.

    • XML로 비교하면

      AnnotationConfigApplicationContext container = new AnnotationConfigApplicationContext(“com.fastcampus.tv”); 이 코드는

      XML에서의 <context:component-scan base-package=”com.fastcampus.tv”/> 과 같다.

    • 스캔: @Configuration, @Service, @Repository, @Controller, @Component

  4. @ComponentScan

    • @ComponentScan(basePackages = {“com.test”, “com.fastcampus.tv”})

  5. @Service

    • @Service(“tv”) / @Service(“sony”) 등으로

  6. 자바소스 설정보다 application.properties 설정이 우선순위가 높다.

    • 위치: src/main/resources/application.properties
    • 설정 내용: 서블릿 실행 유무, 포트, 배너, Bean 오버라이딩, 테스트 설정, DataSource 설정

  7. JSON

  8. @RestController

    • 컨트롤러 설정에 붙는다
    • 컨트롤러 안에있는 인터페이스에 @Autowired가 붙고
    • 컨트롤러 안의 메서드들에 RequestMapping(“/메서드이름”)등으로 매핑한다.

  9. 라이브러리는 Starter(관련된 자료파일 묶음) 를 통해 정의한다. (pom.xml)

    • dependencies 에는 컨트롤 스페이스로 Starter를 삽입할 수 있다.

  10. 메이븐 프로젝트를 만들어서 starer만들기(사용자 정의 starter)

    • 내가 원하는 스타터를 모아서 메이븐 프로젝트로 만들어 사용할 수 있다.
    • 자동설정 스타터는 원하는 프로젝트 pom.xml에 dependency로 삽입할 수 있다.
    • 삽입한 starter가 있다고 하더라도, 오버라이딩 할 수 있다.

JPA

Java Persistence API 로 여러 API를 통합한 API를 이용하면 훨씬 간편하게 프로젝트를 구성할 수 있다.

설정 순서

  1. META-INF에 persistence.xml 설정

    • Datasource와 JPA 구현체(Hibernate)관련 설정 세팅

  2. application.properties 설정

    • 메인모드, 서블릿 구동유무 설정

  3. pom.xml 설정

    • 롬복, h2데이터베이스 등 설정 dependency 추가

  4. @Entity => @Id, @GeneratedValue 설정

    • 비즈니스모델 단위객체 생성 구성요소 (BoardVO)

  5. @Service => @Override

    • 구동할 코드 Overide (INSERT, UPDATE, SELECT, DELETE)
    • SELECT만 제외하고 모두 EntityTransaction 을 사용해야한다.
    • 팩토리생성->컨테이너생성->게시글조회->트랜잭션시작->작업실행 후
    • 트랜잭션 종료->컨테이너 종료->팩토리 종료 순서테스트 관련 3가지

  6. 톰캣 서버 구동 없이 비즈니스 객체 모킹 해서 테스트

    • ??

  7. jar 파일 패키징

    • JPA 프로젝트 우클릭->run as maven install하면 jar파일이 생성된다.
